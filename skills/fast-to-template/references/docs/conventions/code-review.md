# 代码审查规范

使用 `/cr` 审查，支持单 agent 交互式修复和多 agent 对抗式审查。

## 审查优先级

**A 级（正确性与安全性）**：必须通过
- 函数返回值在所有分支中正确设置
- 条件逻辑无 `&&` / `||` 混淆
- Null / undefined 解引用保护
- 异步调用正确 await 和错误处理
- 用户输入未清理不得插入 DOM（`v-html`、`innerHTML`）
- 事件监听器、定时器在组件卸载时清理
- 无硬编码的密钥、令牌或凭据

**B 级（质量与性能）**：建议通过
- 列表渲染使用稳定唯一 key（非数组索引）
- 副作用正确使用生命周期 hooks 和依赖数组
- 组件避免不必要的重渲染（memo / 稳定引用）
- 大型依赖按需导入（tree-shaking）
- 重复逻辑提取，深层嵌套简化为提前返回

**C 级（约定与文档）**：仅标记
- 命名语义清晰，与现有风格一致
- 魔术数字提取为命名常量
- 公共 API 有参数和返回值描述
- 图片有 alt 文本，表单输入有 label

## 排除项

以下不做审查要求：
- 格式化工具已处理的样式问题
- 基于假设的未来需求建议
- 遵循项目现有风格但不匹配外部标准的代码
- 测试代码中的 C 级问题
