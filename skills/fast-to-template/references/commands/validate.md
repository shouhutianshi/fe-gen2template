# 低代码活动页面验证清单

## 验证场景

**输入指令**：`创建一个 SVIP 活动，page=sevenSvip，标题用"SVIP春日特惠"`

---

## 1. JSON 配置校验

### 验证项

| 检查项        | 预期结果                                                                   | 命令                                          |
| ------------- | -------------------------------------------------------------------------- | --------------------------------------------- |
| 配置文件生成  | 生成 `{type}-YYYYMMDD-{index}.json` 格式文件                               | `ls config/pages/*.json`                      |
| JSON 格式正确 | 无语法错误                                                                 | `pnpm lowcode validate config/pages/xxx.json` |
| 必需字段完整  | 含 `$schema`, `version`, `type`, `page`, `title`, `components`, `metadata` | 查看文件内容                                  |

### 必需字段清单

```json
{
  "$schema": "../schemas/activity-config-v1.json",
  "version": "1.0",
  "type": "svip",
  "page": "svip-YYYYMMDD-XX",
  "title": "SVIP春日特惠",
  "components": [...],
  "metadata": { "createdAt": "..." }
}
```

## 5. 常见问题排查

| 问题               | 可能原因       | 解决方案                            |
| ------------------ | -------------- | ----------------------------------- |
| 未知组件: xxx      | 组件未注册     | 检查 `componentRegistry.register()` |
| Hydration mismatch | SSR/CSR 不一致 | 检查组件是否有客户端特有逻辑        |
| CSS MIME 类型警告  | 开发模式 HMR   | 仅开发模式，可忽略                  |
| 组件加载失败       | 导入路径错误   | 检查组件文件是否存在                |

---

## 6. 快速验证命令

```bash
# 1. 创建配置
pnpm lowcode create --type svip --title "SVIP春日特惠"

# 2. 验证配置
pnpm lowcode validate config/pages/svip-*.json

# 3. 开发模式测试
pnpm dev
# 访问 http://localhost:3000/activity

# 4. 生产模式测试
pnpm build && pnpm preview
# 访问 http://localhost:3000/activity
```
