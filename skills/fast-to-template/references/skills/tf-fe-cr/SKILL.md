# /cr — 代码审查

针对本地分支、拉取请求、提交和文件的自动化代码审查。从参数中检测审查模式，并路由到相应的审查流程——要么是快速单智能体审查配合交互式修复选择，要么是基于风险自动修复的多智能体深度审查。

所有面向用户的文本均使用用户的语言。所有问题提问和选项选择必须使用你的交互式对话工具（如 AskUserQuestion）——绝不能以纯文本形式输出选项。在用户回复之前请勿继续执行。

## 路由

运行预检查，然后从上到下匹配**第一条**适用的规则：

1. `git branch --show-current` → 记录是否在 main/master 分支上。
2. `git status --porcelain` → 记录是否存在未提交的更改。
3. 检查当前环境是否支持智能体团队（多个智能体并行工作并相互通信）。

| #   | 条件                                          | 操作                           |
| --- | --------------------------------------------- | ------------------------------ |
| 1   | `$ARGUMENTS` 是 PR 编号或包含 `/pull/` 的 URL | → `references/pr-review.md`    |
| 2   | 不支持智能体团队                              | → `references/local-review.md` |
| 3   | 存在未提交的更改                              | → `references/local-review.md` |
| 4   | 在 main/master 分支上                         | → `references/local-review.md` |
| 5   | 其他所有情况                                  | → 下方问题                     |

每个 `→` 表示：`Read` 读取目标文件并将其作为唯一的剩余指令来执行。忽略以下所有部分。请勿凭记忆或习惯进行审查——每个目标文件都对如何获取差异、应用修复和提交结果定义了特定的约束。

## 问题（仅规则 2）

提出**一个问题**：
"Agent Teams 可用（多个智能体并行工作）。是否启用多智能体审查，采用审查员–验证员对抗机制并自动修复？"
提供 4 个选项：

| 选项                                    | 描述                                                                                 |
| --------------------------------------- | ------------------------------------------------------------------------------------ |
| 单智能体 + 手动修复                     | 单智能体审查；之后交互式选择要修复的问题。                                           |
| 团队 + 自动修复低风险                   | 多智能体审查；仅自动修复最安全的问题（如空值检查、拼写错误、命名）。其他问题需确认。 |
| 团队 + 自动修复低风险和中等风险（推荐） | 多智能体审查；自动修复大多数问题，仅确认高风险问题（如 API 更改、架构）。            |
| 团队 + 自动修复全部                     | 多智能体审查；自动修复所有问题。仅影响测试基准的问题会被延后处理。                   |

### 交接

| 选项                                    | →                            | FIX_MODE   |
| --------------------------------------- | ---------------------------- | ---------- |
| 单智能体 + 手动修复                     | `references/local-review.md` | —          |
| 团队 + 自动修复低风险                   | `references/teams-review.md` | low        |
| 团队 + 自动修复低风险和中等风险（推荐） | `references/teams-review.md` | low_medium |
| 团队 + 自动修复全部                     | `references/teams-review.md` | full       |

将 `$ARGUMENTS` 传递给目标文件。对于 teams-review，还需传递 `FIX_MODE`
（low / low_medium / full）。
